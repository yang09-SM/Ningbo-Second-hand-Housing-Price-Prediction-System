# 宁波二手房房价预测系统 - 迭代升级规格说明

## Why
当前系统已具备基础的房价预测功能（4种模型：线性回归、SVM、随机森林、神经网络），但对比网上同类先进项目（2025-2026年最新实践），在模型精度、可解释性、可视化、用户体验等方面存在显著差距。通过系统性升级，可将预测精度从当前水平提升至 R²>0.90，并增加商业级特性。

## What Changes（按优先级排序）

### 一、模型层升级（高优先级）
- **新增 XGBoost / LightGBM / CatBoost 梯度提升模型** — 研究表明这三种模型在房价预测任务中普遍优于传统随机森林，LightGBM 在测试中 R² 可达 0.99
- **引入 Optuna 自动超参数调优** — 替代手动调参，自动搜索最优超参数组合，提升模型精度 10-30%
- **构建 Stacking 集成学习策略** — 融合多模型预测结果，进一步提升泛化能力
- **添加模型版本管理与 A/B 测试能力** — 支持多版本模型对比和灰度发布

### 二、可解释性模块（高优先级）
- **集成 SHAP 分析引擎** — 提供全局特征重要性（Summary Plot）和局部预测解释（Force Plot / Waterfall Plot）
- **特征归因可视化** — 点击任一预测结果，展示各特征对该预测的正向/负向贡献值
- **支持 TreeExplainer（树模型）/ KernelExplainer（通用模型）** — 兼容不同模型类型的解释需求

### 三、数据可视化大屏（中优先级）
- **构建交互式数据分析仪表盘** — 包含区域房价分布热力图、价格趋势折线图、特征相关性矩阵
- **ECharts / Plotly 动态图表** — 替代静态 matplotlib 图片，支持交互式探索
- **模型性能对比看板** — 多模型指标雷达图、残差分布图、预测 vs 实际散点图

### 四、前端现代化改造（中优先级）
- **UI 组件库升级** — 引入 Element Plus 或 Ant Design，提升界面美观度和交互体验
- **预测表单优化** — 联动下拉选择（区域→小区→户型）、实时输入校验、智能默认值
- **预测结果增强展示** — 价格置信区间、相似房源推荐、影响因素 Top-N 排行

### 五、高级特征工程（中优先级）
- **地理空间特征构造** — 计算距市中心/地铁站/学校/医院的距离特征
- **POI 兴趣点数据融合** — 整合周边配套设施密度作为新特征
- **时序市场特征** — 构建区域均价趋势、成交量变化等时间窗口特征
- **文本特征提取**（可选）— 从房源描述文本中提取装修质量、户型亮点等语义信息

### 六、系统工程化增强（低优先级）
- **RESTful API 规范化** — 统一接口设计，支持批量预测和数据导出
- **预测结果导出功能** — 支持 Excel/PDF 格式的预测报告生成
- **数据管理后台增强** — 支持新成交数据上传、历史数据维护、数据质量监控
- **模型监控与持续学习** — 检测数据漂移、触发模型重训练提醒

## Impact
- Affected specs: `ningbo-house-price-prediction`（现有基础规格）、`user-role-access-control`（用户权限模块）
- Affected code:
  - `src/models/` — 所有模型训练脚本需扩展
  - `src/web/app.py` — 后端 API 需大幅扩展
  - `src/web/templates/index.html` — 前端需重构
  - `reports/` — 报告输出格式需升级
  - 新增 `src/explainability/`（SHAP 模块）
  - 新增 `src/visualization/`（可视化模块）

## ADDED Requirements

### Requirement: 高精度梯度提升模型集成
系统 SHALL 集成 XGBoost、LightGBM、CatBoost 三种梯度提升回归模型，并使用 Optuna 进行超参数自动搜索。

#### Scenario: 模型训练与优选
- **WHEN** 执行模型训练流程
- **THEN** 系统自动完成 XGBoost/LightGBM/CatBoost 的超参搜索与训练，输出各模型在测试集上的 MSE、MAE、R² 指标，并自动选出最优模型用于线上预测

#### Scenario: Stacking 集成预测
- **WHEN** 用户请求房价预测
- **THEN** 系统可选使用 Stacking 策略融合多个基模型的预测结果，输出最终估值及各基模型贡献权重

### Requirement: SHAP 可解释性分析
系统 SHALL 提供 SHAP 值计算与可视化能力，支持全局特征重要性分析和单样本预测归因解释。

#### Scenario: 全局特征重要性查看
- **WHEN** 用户访问模型分析页面
- **THEN** 展示 SHAP Summary Plot（蜂群图/条形图），显示所有特征对预测结果的平均影响程度和方向

#### Scenario: 单样本预测解释
- **WHEN** 用户完成一次房价预测并点击"查看解释"
- **THEN** 展示 SHAP Force Plot / Waterfall Plot，以颜色（红升/蓝降）和长度直观呈现每个特征将该样本预测值推高或拉低的幅度

### Requirement: 交互式数据可视化大屏
系统 SHALL 提供基于 Web 的交互式数据分析和可视化面板。

#### Scenario: 数据概览仪表盘
- **WHEN** 用户进入系统首页
- **THEN** 展示宁波各区房价热力图、价格分布直方图、关键指标卡片（均价、中位数、房源量、模型 R²），所有图表支持缩放、筛选、悬停提示等交互操作

#### Scenario: 模型性能对比看板
- **WHEN** 用户进入模型分析页面
- **THEN** 以雷达图/柱状图对比所有模型的评估指标，以散点图展示预测值 vs 实际值分布，以残差图展示模型误差模式

### Requirement: 前端用户体验升级
系统 SHALL 使用现代前端组件库重构用户界面，提供流畅的交互体验。

#### Scenario: 智能预测表单
- **WHEN** 用户填写房屋信息进行预测
- **THEN** 表单支持联动选择（如选择区域后动态加载小区列表）、输入实时校验、智能填充默认值，提交后展示带置信区间的预测结果及 Top-5 影响因素排行

### Requirement: 高级特征工程管道
系统 SHALL 扩展特征工程流程，支持地理距离特征、POI 密度特征及时序市场特征的自动构造。

#### Scenario: 特征自动构造
- **WHEN** 执行数据预处理流程
- **THEN** 系统自动计算每套房源到市中心/最近地铁/重点学校的距离特征，统计周边 1km/3km 范围内 POI 数量，并基于历史成交数据构建区域价格趋势特征

## MODIFIED Requirements

### Requirement: 模型评估与对比（原 FR-7）
修改为：系统 SHALL 支持不少于 7 种模型的训练与评估（原 4 种 + XGBoost + LightGBM + CatBoost），评估指标除 MSE/MAE/R² 外，新增 RMSE、MAPE、SHAP 一致性指标；评估报告 SHALL 包含 SHAP 可视化图表。

### Requirement: Web 管理界面（原 FR-12）
修改为：系统 SHALL 提供包含以下模块的 Web 管理界面：
1. 数据可视化大屏（首页仪表盘）
2. 房源数据管理（增删改查 + 搜索筛选）
3. 智能房价预测（增强表单 + 结果解释）
4. 模型分析中心（性能对比 + SHAP 解释）
5. 系统管理后台（用户权限 + 数据维护）

## REMOVED Requirements
无。所有原有功能保留，仅在其基础上进行增量升级。
